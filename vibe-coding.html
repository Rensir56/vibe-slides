🔬 原理探索

AI 代码生成是怎么工作的？

ReAct = Reason + Action

让 AI 边思考边行动，形成推理-行动的循环

1

理解需求

AI 阅读你的描述，理解要做什么

2

规划步骤

思考需要哪几步来完成，每个步骤怎么做

3

生成代码

根据计划写出具体代码，并检查是否正确

💡 简单理解：AI 就像是有一个"思考过程"，不是凭空写代码，而是一边想"这一步该做什么"，一边写代码

🧰 Claude Code Skill

Skill 是什么？

Skill = 给 Claude Code 安装一套 可复用的做事方法

当你总是在复制同一段提示词、同一份检查清单、同一个工作流程时，就可以把它沉淀成 skill。

📌

固定流程

把“先检查、再修改、再测试”的步骤写进 SKILL.md，减少每次解释

🧠

按需加载

Skill 的正文只有用到时才进入上下文，比把所有规则塞进 CLAUDE.md 更轻

⚡

一键调用

可以输入 /skill-name 直接触发，也可以让 Claude 在相关场景自动使用

📦 获取方式

方式 1

方式 2

📝 基本结构

真实 Skill 文件结构

一个 skill 是一个目录

官方结构强调“渐进加载”：先看 metadata，触发后读 SKILL.md，需要时再读其它文件。

my-skill/ ├── SKILL.md ├── references/ # 长文档，按需读取 ├── scripts/ # 稳定脚本，按需执行 └── assets/ # 模板/素材，给最终产物使用

SKILL.md 是入口文件

--- name: summarize-changes description: Use when the user asks to summarize current git changes. disable-model-invocation: true allowed-tools: Bash(git status *) Bash(git diff *) --- ## Instructions 1. Run git status and git diff. 2. Summarize changed files and behavior changes. 3. List risks and missing tests.

YAML metadata

name 是调用名；description 是触发条件，要写清“什么时候用”。权限字段限制 skill 可以调用哪些工具。

工具权限

allowed-tools 像白名单，限制这个 skill 能跑哪些命令。提交、删除、部署这类动作尤其要收紧。

Markdown 正文

写工作步骤、检查清单、输出格式和禁止事项。正文应短，把模型已经知道的常识删掉。

附加资源

references 放大段资料，scripts 放可重复执行的逻辑，assets 放模板素材。

🤖 AI 生成

让 AI 帮你生成 Skill

Codex

直接调用 skill-creator

输入 $ skill-creator 或直接说“帮我创建一个 skill”。Codex 会按规范生成目录、SKILL.md 和必要资源。

Claude Code

让 cc 写入 skills 目录

官方方式是创建 .claude/skills/name/SKILL.md 或 ~/.claude/skills/name/SKILL.md，再用 /name 调用。

给 AI 的提示词可以这样写：

把下面流程做成一个 skill，放到 .claude/skills/commit/SKILL.md： 1. 写完代码后先运行测试 2. 再查看 git status 和 git diff 3. 总结改动、风险和缺失测试 4. 只有我明确要求时才 git commit 请控制 allowed-tools，并说明如何用 /commit 调用

目标

这个 skill 是用来审代码、写提交、部署，还是生成文档？

边界

哪些命令能跑，哪些动作必须先问你，哪些文件不能改？

验证

创建后用 /commit 试运行；确认它先检查 diff，不会在未授权时自动提交。

🧪 案例：自动提交

写完代码后直接 git commit

--- name: commit description: Stage and commit the current changes after reviewing the diff. disable-model-invocation: true allowed-tools: Bash(git status *) Bash(git diff *) Bash(git add *) Bash(git commit *) --- 1. Run git status and git diff. 2. Summarize the changes and mention any risk. 3. If the diff looks ready, run git add for the relevant files. 4. Create a concise commit message and run git commit.

为什么手动触发？

commit 有副作用，用 disable-model-invocation: true，避免 AI 自己决定提交

为什么先看 diff？

提交前必须知道改了什么，避免把调试代码、密钥、无关文件一起提交

怎么调用？

代码写完后输入 /commit，让 Claude 按固定流程检查并提交

🧩 Skills 扩展

Claude Code 内置 Skills

什么是 Skill？Skill 是 Claude Code 内置的特殊命令，用 /skill-name 触发，提供特定领域的专业能力

/simplify

审查代码并提出优化建议，重构冗余部分

/loop

设置定时任务，定期检查状态或运行命令

/init

/plan

💡 技巧 1/3

描述要具体

❌ 不好的描述

"帮我写个按钮"

✅ 好的描述

"写一个圆形红色按钮，边框圆角 20px，点击后弹出 '提交成功' 提示"

🎯 为什么重要？

越具体的描述，AI 越能准确理解你的需求，减少返工次数

💡 技巧 2/3

分步骤进行

不要试图一次性完成整个项目

Step 1 先做页面结构 → Step 2 添加样式 → Step 3 实现交互

🌱 小步快跑

每一步都能看到成果，发现问题及时调整

🔍 便于调试

出错了能快速定位是哪一步的问题

💡 技巧 3/3

检查并修改

⚠️ AI 生成的代码需要你审核

AI 可能会生成有 bug 的代码，或者实现和你想象的不一样。每次生成后，先运行看看效果，发现问题立刻告诉 AI 怎么改。

常用修改指令：

"把这个按钮变大" "颜色改成绿色" "添加点击效果"

⚠️ 注意事项

新手常见的错误

❌ 完全不看代码

AI 写的代码不一定对，必须自己读一遍理解

❌ 一次说太多

"帮我做一个抖音"，这种需求 AI 无法准确完成

❌ 不检查直接运行

先看懂代码逻辑，确认没问题再运行

❌ 遇到问题不描述

出错了把错误信息复制给 AI，它会帮你分析
