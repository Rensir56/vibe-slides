🔬 原理探索

AI 代码生成是怎么工作的？

ReAct = Reason + Action

让 AI 边思考边行动，形成推理-行动的循环

1

理解需求

AI 阅读你的描述，理解要做什么

2

规划步骤

思考需要哪几步来完成，每个步骤怎么做

3

生成代码

根据计划写出具体代码，并检查是否正确

💡 简单理解：AI 就像是有一个"思考过程"，不是凭空写代码，而是一边想"这一步该做什么"，一边写代码

🔧 工具介绍

Cursor - 智能代码编辑器

🎯

AI 原生 IDE

专为 AI 协作设计的代码编辑器，基于 VS Code

💬

对话式编程

侧边栏对话，直接描述需求，AI 帮你写代码

⚡

实时补全

输入时 AI 自动预测下一行代码

🔍 核心功能

Cursor 的三大核心功能

1

Ctrl+L 对话

打开 AI 对话窗口，输入自然语言描述需求，AI 直接在代码中修改

2

Tab 自动补全

AI 预测下一段代码，按 Tab 直接采纳，比 GitHub Copilot 更懂上下文

3

Ctrl+K 代码生成

选中代码块，告诉 AI 你想怎么改，比如"把这个函数改成异步"

🧪 实操演示

用 Cursor 做一个按钮

在 Cursor 中打开一个 HTML 文件，然后输入：

/* 在 Ctrl+L 对话框中输入： */ "给我写一个蓝色按钮，点击弹出 'Hello World'"

✨ AI 会自动生成：

<button style="background: blue; color: white; padding: 10px 20px;" onclick="alert('Hello World')"> 点击我 </button>
💻 工具介绍

Claude Code - 命令行助手

🖥️

终端内直接使用

在命令行中输入需求，AI 自动创建文件、修改代码、运行测试

📁

项目级理解

能够阅读整个项目代码，理解项目结构和依赖关系

🔗

Git 集成

自动提交代码、创建分支、查看 diff，处理 Git 工作流

⚙️ 工作流程

Claude Code 典型工作流程

Step 1 npx claude-code
Step 2 描述你要做什么："帮我创建一个 React 组件"
Step 3 Claude 自动创建/修改文件，可直接运行
Step 4 检查结果，不满意就说"改一下..."
🧪 实操演示

用 Claude Code 创建项目

在终端输入：

$ npx claude-code $ "创建一个 HTML 页面，包含一个输入框和一个按钮"

✨ Claude 会：

1. 创建 index.html 文件
2. 写入完整的 HTML 代码
3. 询问是否要打开浏览器预览

💡 技巧 1/3

描述要具体

❌ 不好的描述

"帮我写个按钮"

✅ 好的描述

"写一个圆形红色按钮，边框圆角 20px，点击后弹出 '提交成功' 提示"

🎯 为什么重要？

越具体的描述，AI 越能准确理解你的需求，减少返工次数

💡 技巧 2/3

分步骤进行

不要试图一次性完成整个项目

Step 1 先做页面结构 → Step 2 添加样式 → Step 3 实现交互

🌱 小步快跑

每一步都能看到成果，发现问题及时调整

🔍 便于调试

出错了能快速定位是哪一步的问题

💡 技巧 3/3

检查并修改

⚠️ AI 生成的代码需要你审核

AI 可能会生成有 bug 的代码，或者实现和你想象的不一样。每次生成后，先运行看看效果，发现问题立刻告诉 AI 怎么改。

常用修改指令：

"把这个按钮变大" "颜色改成绿色" "添加点击效果"
🧩 练习 1

案例：做一个计数器

需求描述：

/* 输入： */ "创建一个页面，中间显示数字 0，底部有 '+' 和 '-' 两个按钮， 点击 '+' 数字加 1，点击 '-' 数字减 1"

你会学到的：

HTML 结构 CSS 布局 JavaScript 事件
🧩 练习 2

案例：做一个待办清单

需求描述：

/* 输入： */ "创建一个待办事项页面，上面是一个输入框和一个添加按钮， 下面是一列表格，每行显示待办事项，最右边有删除按钮"

你会学到的：

数组操作 DOM 操作 数据持久化
⚠️ 注意事项

新手常见的错误

❌ 完全不看代码

AI 写的代码不一定对，必须自己读一遍理解

❌ 一次说太多

"帮我做一个抖音"，这种需求 AI 无法准确完成

❌ 不检查直接运行

先看懂代码逻辑，确认没问题再运行

❌ 遇到问题不描述

出错了把错误信息复制给 AI，它会帮你分析
